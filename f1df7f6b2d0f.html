    【Divino 的诞生】实现基于 etcd 的 gRPC 服务注册与发现

    Divino 本来使用的是 Consul 作为服务注册与发现中心，但是随着 gRPC 移除了 naming 包，使用 Consul 作为服务注册与发现中心变得十分困难，而此时我发现 etcd 提供了官方的 gRPC Naming Resolver，于是就将服务注册与发现功能迁移到了 etcd，在此也顺便讲讲 naming.Resolver 被移除之后需要如何实现服务注册与发现功能。

    服务注册与发现的基本流程

    基于 etcd 的 gRPC 服务注册与发现的基本流程如下：

    1. 初始化服务注册与发现客户端（其中包含一个空的服务实例列表、用于控制服务注册与发现过程的上下文 context.Context、etcd 客户端等）；
    2. 初始化服务元数据，包含服务名称、服务编号（唯一，可用 uuid 生成）、gRPC 监听的地址等，并将该元数据包装成 etcd gRPC Resolver 可以识别的格式：{Addr: ..., Metadata: {...}}（也就是 endpoint.Endpoint）；
    3. 申请一个租约（lease）用于健康检查；
    4. 向 etcd PUT 一个键值对，其中 key 具有一定的格式（Divino使用的格式是 /divino_services/服务名称/服务编号，这主要是为了与 etcd 中的其他数据进行区分），value 为第 2 步初始化的服务元数据；
    5. 使用 KeepAlive 配合 lease 实现健康检查；
    6. 监听某一个前缀的键值对变化，动态地更新服务实例列表；
    7. 监听退出信号，在收到信号之后删除 etcd 中对应的键值对，取消服务注册。

    元数据的结构设计

    元数据除了要包含服务的一些信息（服务名称、服务编号、gRPC 监听的地址等）还要可扩展，以便携带一些可供其他微服务使用的信息。最终根据需求，设计了以下格式的元数据：

    // ServiceInfo 服务注册信息（服务元数据）
type ServiceInfo struct {
    ID      string                 // 服务实例编号
    Name    string                 // 服务名称
    Address string                 // 服务地址
    Tags    map[string]interface{} // 服务注册时所携带的附加信息
}

    当然，为了让这些信息可以被 etcd gRPC Naming Resolver 识别，还需要再包装一层：

    // ServiceInfoSer 服务注册信息的 etcd 表示形式
type ServiceInfoSer struct {
    Addr     string      `json:"Addr"`     // 服务地址
    Metadata ServiceInfo `json:"Metadata"` // 服务元数据（ServiceInfo）
}

    这其实就是 endpoint.Endpoint，也有的文章是使用 Endpoint 配合 EndpointManager 实现 gRPC 服务注册与发现的，这样在代码方面会简单一些，但是考虑到除了 gRPC 服务注册与发现之外，其他一些功能也需要使用服务注册与发现的这些元数据（例如 divino-server 的动态服务发现列表展示功能等），所以就没有使用 endpoint.Endpoint，而是自己定义了相同的数据结构并且没有使用 EndpointManager。

    服务注册与发现客户端的设计

    一个服务注册与发现客户端应该包括：

    1. etcd client 实例，用于操作 etcd；
    2. context.Context，用于控制服务注册与发现过程的执行（如果这个 Context 被 cancel 了，那么 etcd 操作也会一并被 cancel）；
    3. 一个服务实例列表，这里为了方便查询，设计成了双层 map：{key: 服务名称, value: {key: 服务编号, value: 服务元数据}}。
    // ServiceDiscoveryClient 服务注册与发现客户端
type ServiceDiscoveryClient struct {
    client   *clientv3.Client
    ctx      context.Context
    services map[string]map[string]*ServiceInfo // 外层 map key 为服务名，内层 map key 为 ID
}

    此时也可以写出初始化服务注册与发现客户端的逻辑了：

    // NewDiscoveryClient
// @Description 初始化服务注册与发现客户端
// @param ctx context.Context
// @param endpoints etcd 地址，可以有多个
// @param dialTimeout 超时时间
// @return *ServiceDiscoveryClient 初始化成功时返回一个新的服务注册与发现客户端，失败时返回 nil
// @return error 初始化成功时返回 nil，失败时返回具体的错误信息
func NewDiscoveryClient(ctx context.Context, endpoints string, dialTimeout time.Duration) (*ServiceDiscoveryClient, error) {
    // 初始化 etcd 客户端
    config := clientv3.Config{
        Endpoints:   strings.Split(endpoints, ","),
        DialTimeout: dialTimeout,
    }
    cli, err := clientv3.New(config)
    if err != nil {
        return nil, err
    }
    // 构造成服务注册与发现客户端，返回
    return &ServiceDiscoveryClient{
        client:   cli,
        ctx:      ctx,
        services: make(map[string]map[string]*ServiceInfo),
    }, nil
}

    根据以上设计，配合前文提到的关于 etcd key 的约定，最基本的向服务实例列表中添加或者删除服务实例的功能就可以实现了：

    // addService
// @Description 为 ServiceDiscoveryClient 实例列表增加服务信息
// @param key 新发现的服务在注册中心的 key
// @param info 新发现的服务的 ServiceInfo，需要保存到 services 字段的 map 中
func (c *ServiceDiscoveryClient) addService(key []byte, info ServiceInfo) {
    reg := regexp.MustCompile(`/divino_services/(\w+)/(.*)`)
    if reg.Match(key) {
        keyInfo := reg.FindSubmatch(key)
        // keyInfo[0]：整个 Key
        // keyInfo[1]：服务名称
        // keyInfo[2]：服务实例编号
        serviceName := string(keyInfo[1])
        serviceID := string(keyInfo[2])
        // 将发现的服务保存到实例列表中
        if c.services[serviceName] == nil {
            // 如果该服务对应的 map 还没有建立，就初始化一个
            c.services[serviceName] = make(map[string]*ServiceInfo)
        }
        // 否则直接添加到 map 对应的表项中
        c.services[serviceName][serviceID] = &info
    }
    logger.Logger.Infof("[Discovery] Service discovered: %v", string(key))
}

// deleteService
// @Description 删除 ServiceDiscoveryClient 实例列表中的某个服务实例
// @param key 服务实例在服务注册中心的 key
func (c *ServiceDiscoveryClient) deleteService(key []byte) {
    reg := regexp.MustCompile(`/divino_services/(\w+)/(.*)`)
    if reg.Match(key) {
        keyInfo := reg.FindSubmatch(key)
        serviceName := string(keyInfo[1])
        serviceID := string(keyInfo[2])
        // 取消注册的情况下直接删除 service 表对应的服务名称的 map 里对应的服务实例即可
        delete(c.services[serviceName], serviceID)
        // 如果该服务在注册中心里已经没有实例了，就直接删掉该服务名称对应的 map
        if len(c.services[serviceName]) == 0 {
            delete(c.services, serviceName)
        }
    }
    logger.Logger.Infof("[Discovery] Service deleted: %v", string(key))
}

    可以看到用双层 map 大大方便了删除功能的编写，只需要使用两个 delete 函数就能完成删除功能。

    服务注册与发现功能的实现

    有了服务注册与发现客户端和NewDiscoveryClient、addService、deleteService 这三个基本的函数之后，服务注册与发现功能就很好实现了。由于我的注释写的比较详细，大家看注释来理解就可以了：

    服务注册：

    // Register
// @Description 注册服务
// @param info 服务的信息，包含服务名称、服务 ID、服务地址等
// @param watchPrefix 要监听的服务前缀，设置这个参数是为了方便指定需要监听哪些服务实例列表的变化
// @return error 注册失败时返回注册失败的错误信息
// @return error
func (c *ServiceDiscoveryClient) Register(info ServiceInfo, watchPrefix []string) error {
    // 服务注册格式：/divino_services/服务名称/服务实例编号
    key := fmt.Sprintf("%s/%s/%s", CommonPrefix, info.Name, info.ID)
    kv := clientv3.NewKV(c.client)
    lease := clientv3.NewLease(c.client)

    // 设置 60 秒租约，也就是 60 秒健康检查一次
    leaseResp, err := lease.Grant(c.ctx, 60)
    if err != nil {
        return err
    }

    // 初始化服务注册信息，注册到 etcd
    regInfo := ServiceInfoSer{
        Addr:     info.Address,
        Metadata: info,
    }
    b, err := json.Marshal(regInfo)
    if err != nil {
        return err
    }
    _, err = kv.Put(c.ctx, key, string(b), clientv3.WithLease(leaseResp.ID))
    if err != nil {
        return err
    }
    logger.Logger.Infof("[Discovery] Service registered, key: %v", key)

    // KeepAlive 配合 lease 使用，实现健康检查的功能
    ch, err := c.client.KeepAlive(c.ctx, leaseResp.ID)
    if err != nil {
        logger.Logger.Errorf("[Discovery] Failed to KeepAlive: %v", err.Error())
        return err
    }
    go func() {
        // 不断取出并丢掉 KeepAlive channel 里面的值，防止出现 "lease keepalive response queue is full; dropping response send"
        for {
            _ = <-ch
        }
    }()

    // 监听服务实例变化，动态更新本地服务发现实例列表
    for _, p := range watchPrefix {
        go c.watch(p)
    }

    return nil
}

    取消注册：

    // UnRegister
// @Description 取消服务注册
// @param info 服务的信息，这里要用到服务 ID 和服务名称
// @return error 取消注册失败时返回取消注册失败的错误信息
func (c *ServiceDiscoveryClient) UnRegister(info ServiceInfo) error {
    // 取消服务注册时，只需要删除注册中心里对应的键值对即可
    key := fmt.Sprintf("%s/%s/%s", CommonPrefix, info.Name, info.ID)
    kv := clientv3.NewKV(c.client)
    _, err := kv.Delete(c.ctx, key)
    if err != nil {
        return err
    }
    logger.Logger.Infof("[Discovery] Service unregistered, key: %v", key)
    return nil
}

    监听 etcd 并动态更新服务实例列表：

    // watch
// @Description 监听 etcd 中某个前缀下的服务变化，动态更新服务实例列表
// @param prefix 要监听的前缀
func (c *ServiceDiscoveryClient) watch(prefix string) {
    // 监听前缀为 prefix 的键值对的变化
    watchChan := c.client.Watch(c.ctx, prefix, clientv3.WithPrefix())
    logger.Logger.Infof("[Discovery] Watching prefix: %v", prefix)
    // 阻塞点，每当 etcd 中的服务实例列表发生变化时，这里才会继续往下运行
    for watchResp := range watchChan {
        for _, e := range watchResp.Events {
            switch e.Type {
            case mvccpb.PUT: // 新增服务
                var serviceInfo ServiceInfoSer
                err := json.Unmarshal(e.Kv.Value, &serviceInfo)
                if err != nil {
                    logger.Logger.Errorf("[Discovery] Failed to unmarshal ServiceInfoSer, skipping: %v", err)
                } else {
                    c.addService(e.Kv.Key, serviceInfo.Metadata)
                }
            case mvccpb.DELETE: // 删除服务
                c.deleteService(e.Kv.Key)
            }
        }
    }
}

    package main

import (
    "context"
    "flag"
    "fmt"
    "github.com/gin-gonic/gin"
    "github.com/google/uuid"
    "github.com/wendev/divino/divino-common/api"
    "github.com/wendev/divino/divino-common/discovery"
    "github.com/wendev/divino/divino-common/logger"
    "google.golang.org/grpc"
    "net"
    "net/http"
    "os"
    "os/signal"
    "syscall"
    "time"
)

var grpcAddress string
var httpAddress string

func init() {
    flag.StringVar(&grpcAddress, "grpc-address", "127.0.0.1:8131", "divino_server gRPC 监听的地址")
    flag.StringVar(&httpAddress, "http-address", "127.0.0.1:8134", "divino_server HTTP 监听的地址")
    if !flag.Parsed() {
        flag.Parse()
    }
}

func main() {
    // 初始化服务注册与发现客户端
    discoveryClient, err := discovery.NewDiscoveryClient(context.Background(), "127.0.0.1:2379", 10*time.Second)
    if err != nil {
        logger.Logger.Fatal(err)
    }
    // 初始化服务元数据
    serviceInfo := discovery.ServiceInfo{
        ID:      uuid.New().String(),
        Name:    "divino_server",
        Address: grpcAddress,
        Tags:    nil,
    }
    // 服务注册
    err = discoveryClient.Register(serviceInfo, []string{discovery.CommonPrefix + "/divino_proxy"})
    if err != nil {
        logger.Logger.Fatal(err)
    }

    go func() {
        // 启动 gRPC 服务端
        lis, err := net.Listen("tcp", grpcAddress)
        if err != nil {
            logger.Logger.Fatalf("Failed to listen: %v", err)
        }
        fmt.Println("listening on " + grpcAddress)
        server := grpc.NewServer()
        api.RegisterTestServer(server, &TestRPCServer{})
        if err := server.Serve(lis); err != nil {
            logger.Logger.Fatal(err)
        }
    }()

    app := gin.Default()
    server := &http.Server{
        Addr:    httpAddress,
        Handler: app,
    }

    go func() {
        // 开启 gin http 服务
        if err := server.ListenAndServe(); err != nil && err != http.ErrServerClosed {
            logger.Logger.Fatalf("Failed to listen: %v", err.Error())
        }
    }()

    // 监听中止信号，实现 graceful shutdown
    quit := make(chan os.Signal)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM, syscall.SIGQUIT)
    <-quit
    logger.Logger.Info("Shutting down server...")
    // 取消服务注册
    err = discoveryClient.UnRegister(serviceInfo)
    if err != nil {
        logger.Logger.Fatalf("Failed to unregister: %v", err.Error())
    }
    // 关闭 http 服务
    ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
    defer cancel()
    if err := server.Shutdown(ctx); err != nil {
        logger.Logger.Fatalf("Failed to shut down server: %v", err.Error())
    }
    logger.Logger.Infof("Server exited.")
}

type TestRPCServer struct {
    api.TestServer
}

func (s *TestRPCServer) TestRPC(ctx context.Context, req *api.TestRequest) (*api.TestResponse, error) {
    return &api.TestResponse{Message: "Hello, " + req.Name + ", server grpcAddress is " + grpcAddress}, nil
}

    defer cancel()
    for i := 0; i < 5; i++ {
        response, err := cli.TestRPC(ctx, &api.TestRequest{Name: "divino-proxy"})
        if err != nil {
            panic(err)
        }
        fmt.Println(response.Message)
    }

    // 调用完成后，取消服务注册
    err = discoveryClient.UnRegister(serviceInfo)
    if err != nil {
        logger.Logger.Fatalf("Failed to unregister: %v", err.Error())
    }
}

func NewClient(disClient *discovery.ServiceDiscoveryClient) (*grpc.ClientConn, error) {
    etcdResolver, err := resolver.NewBuilder(disClient.GetEtcdClient())
    // 这里第二个参数要写需要调用的服务名，不是自己的服务名哈
    addr := fmt.Sprintf("etcd:///%s/%s", discovery.CommonPrefix, "divino_server")
    conn, err := grpc.Dial(addr, grpc.WithTransportCredentials(insecure.NewCredentials()), grpc.WithDefaultServiceConfig(fmt.Sprintf(`{"LoadBalancingPolicy": "%s"}`, roundrobin.Name)), grpc.WithResolvers(etcdResolver))
    if err != nil {
        return nil, err
    }
    return conn, nil
}

    总结

    服务注册与发现可以说是微服务架构的系统中最重要的基础功能之一了，也是服务之间互相通信的基础。另外，动态配置管理、 etcd 分布式锁等功能的原理其实跟服务注册与发现差不多，其本质上也是对 etcd 中的键值对的操作与监听和对 lease 的灵活使用，如果能完全理解基于 etcd 的服务发现的原理即实现方法，那么这些功能的实现对你来说肯定也不复杂。
