    733. 图像渲染

    题目地址：733. 图像渲染

    解法1：BFS

    首先将起点更改颜色后入队，然后当队列不空时，每次将一个像素点出队，然后将其周围的四个像素点更改颜色并入队（注意边界条件的判断，只有在图像范围内的像素点才能入队），队列为空时算法结束。

    // r,c代表与当前像素点相邻的四个像素点与当前像素点的坐标之差
var (
    r = []int{1, 0, 0, -1}
    c = []int{0, -1, 1, 0}
)

type pixel struct {
    r, c int
}

func floodFill(image [][]int, sr int, sc int, newColor int) [][]int {
    curColor := image[sr][sc]
    if curColor == newColor {
        return image
    }
    // 得到图像的宽和高
    w, h := len(image[0]), len(image)
    queue := make([]pixel, 0)
    // 将像素的初始位置更改颜色并加入队列
    queue = append(queue, pixel{r: sr, c: sc})
    image[sr][sc] = newColor
    // 开始BFS算法
    for len(queue) > 0 {
        // 每次将一个像素点出队
        p := queue[len(queue)-1]
        queue = queue[:len(queue)-1]
        // 处理该像素相邻的四个像素点
        for i := 0; i < 4; i++ {
            rr, cc := p.r + r[i], p.c + c[i]
            if rr >= 0 && cc >= 0 && rr < h && cc < w && image[rr][cc] == curColor {
                queue = append(queue, pixel{r: rr, c: cc})
                image[rr][cc] = newColor
            }
        }
    }
    return image
}

    内存消耗：4.2 MB, 在所有 Go 提交中击败了31.90%的用户

    时间复杂度：O(m * n)，二维数组中的每一个位置（像素）都会被访问一次；

    空间复杂度：O(m * n)，来自队列。

    解法2：DFS

    本题同样可以使用DFS来解。首先使用变量curColor保存当前位置的颜色，然后进行DFS操作。每搜索到一个像素时，如果其颜色与curColor相同，就将其更新为newColor，然后再对其相邻的四个方格递归地进行渲染操作。

    // r,c代表与当前像素点相邻的四个像素点与当前像素点的坐标之差
var (
    r = []int{1, 0, 0, -1}
    c = []int{0, -1, 1, 0}
)

func floodFill(image [][]int, sr int, sc int, newColor int) [][]int {
    curColor := image[sr][sc]
    if curColor != newColor {
        dfs(image, sr, sc, curColor, newColor)
    }
    return image
}

func dfs(image [][]int, rr, cc, curColor, newColor int) {
    if image[rr][cc] == curColor {
        image[rr][cc] = newColor
        for i := 0; i < 4; i++ {
            rrr, ccc := rr+r[i], cc+c[i]
            if rrr >= 0 && rrr < len(image) && ccc >= 0 && ccc < len(image[0]) {
                dfs(image, rrr, ccc, curColor, newColor)
            }
        }
    }
}

    执行用时：12 ms, 在所有 Go 提交中击败了5.22%的用户；

    内存消耗：4.1 MB, 在所有 Go 提交中击败了75.54%的用户；

    时间复杂度：O(m * n)，二维数组的每个位置（像素）都会被访问一次；

    空间复杂度：O(m * n)，来自递归工作栈。
