    2.两数相加

    2. 两数相加

    题目地址：2. 两数相加

    解法：模拟加法

    很容易就能看出这题给出的链表长度已经远远超出了go语言中最大整数数值类型uint64所能表示的范围，所以需要对两个链表做模拟加法运算才能得出正确答案。

    在两个链表都没有遍历完时，这个过程就是一个很简单的模拟大数加法而已；其中一个链表遍历完之后，对另一个链表继续使用模拟加法进行计算，当然我们可能还会遇到两个链表都遍历完之后仍有进位没有消除的情况，这时再挂一个1到链表上表示进位即可。这里使用的是带有头结点的链表，所以最后返回答案时需要摘除头结点（即返回head.Next）。

    /**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func addTwoNumbers(l1 *ListNode, l2 *ListNode) *ListNode {
    p1, p2 := l1, l2
    // 头结点,没啥用,只是为了方便创建链表,创建完之后head.Next才是我们要的答案
    head := &ListNode{Val: 0, Next: nil}
    p := head
    c := 0
    for p1 != nil && p2 != nil {
        v := p1.Val + p2.Val
        // 判断上一位有无进位,如果有,则给这一位的数加1并且清除进位标志
        if c == 1 {
            v += 1
            c = 0
        }
        // 判断该位置上的两个数相加结果有没有超过10,超过则进位
        if v >= 10 {
            c = 1
        }
        // 新建一个链表结点,把这个数存进去,并把结点挂在链表上
        node := &ListNode{Val: v % 10, Next: nil}
        p.Next = node
        // 三个指针往后走一个结点
        p = p.Next
        p1 = p1.Next
        p2 = p2.Next
    }
    if p1 != nil {
        for p3 := p1; p3 != nil; p3 = p3.Next {
            v := p3.Val
            // 判断上一位有无进位,如果有,则给这一位的数加1并且清除进位标志
            if c == 1 {
                v += 1
                c = 0
            }
            // 判断该位置上的两个数相加结果有没有超过10,超过则进位
            if v >= 10 {
                c = 1
            }
            node := &ListNode{Val: v % 10, Next: nil}
            p.Next = node
            p = p.Next
        }
    } else if p2 != nil {
        for p4 := p2; p4 != nil; p4 = p4.Next {
            v := p4.Val
            // 判断上一位有无进位,如果有,则给这一位的数加1并且清除进位标志
            if c == 1 {
                v += 1
                c = 0
            }
            // 判断该位置上的两个数相加结果有没有超过10,超过则进位
            if v >= 10 {
                c = 1
            }
            node := &ListNode{Val: v % 10, Next: nil}
            p.Next = node
            p = p.Next
        }
    }
    // 看看两个链表遍历完之后还有没有进位,有的话就再挂一个1上去
    if c == 1 {
        node := &ListNode{Val: 1, Next: nil}
        p.Next = node
        p = p.Next
        c = 0
    }
    return head.Next
}

    执行用时: 16 ms（超过57%的Golang提交记录）

    内存消耗: 4.6 MB（超过91%的Golang提交记录）

    时间复杂度：O(max(m, n))，循环的次数取决于两个链表中较长的一个；

    空间复杂度：O(1)，除了返回的链表之外只使用了常数个数的存储空间。
